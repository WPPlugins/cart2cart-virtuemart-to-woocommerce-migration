Use this plugin to move your data from VirtueMart to WooCommerce rapidly and with no efforts.

== Description ==
**Supported WooCommerce versions: 1.1.x – 1.6.x, 2.x, 3.x**
**Supported VirtueMart versions: 1.x, 2.х, 3.x**

A plugin will help you to switch from your VirtueMart store to WooCommerce in a couple of hours. Your products, customers and orders will be migrated really easily. Pay attention that no coding or scripting experience is necessary.

**The plugin offers 2 migration options**

1. Free Demo migration — lets you perform migration of products, orders, customers and categories in limited quantity (up to 10 entities each) for test purposes.
1. Full Migration — lets you perform UNLIMITED migration of products, orders, customers, categories, variants, attributes and other related entities. Full Migration price starts at $69 and may vary depending on the number of transferable entities and additional options chosen.

= Features of automated migration =
- **Simple** - follow you migration procedure via straightforward wizard`s interface.
- **Fast** - migration takes only a couple of hours.
- **Free Demo** - try free migration before you perform paid one.
- **Support** - you can always reach customer service when you have any questions.

[youtube https://www.youtube.com/watch?v=xmq7u9X9XUs]

= You can migrate such data from VirtueMart to WooCommerce: =
* Products, product images, product attributes;
* Orders, order statuses, order products;
* Customers, shipping address, billing address;
* Categories, category images;
* Additional: currencies, manufacturers, manufacturer images, taxes.

**Please note.** *User passwords are not migrated due to encryption. Migration includes only entities move.*

This plugin performs the installation of connection bridges in order to do Cart2Cart migration.
Migration bridges allow safe data transfer between your stores.When they are installed, you will be redirected to Cart2Cart migration wizard.


= Steps to take before migration =
* Install your VirtueMart store, note that both of your webstore have to be online.
* Prepare valid FTP/admin logins and passwords for your VirtueMart.  You will need them to install connection bridge.

==Installation==
1. Download the plugin archived file.
1. Perform an extraction of plugin zip file to your computer.
1. Move unarchived files to wp-content/plugin directory.
1. Proceed to Admin -> Plugins, click Activate “Cart2Cart VirtueMart to WooCommerce Migration”.
1. Insert OpenCart  FTP details and download a connection bridge.
1. WooCommerce connection bridge will be uploaded automatically.
1. Click Start Migration. Afterwards you are going to be  redirected to Cart2Cart website to continue your switch.

== Frequently Asked Questions ==
= What is Cart2Cart? =
Cart2Cart is a shopping cart migration service which allows automated data transfer from one platform to another. Currently service supports migration from/to [70+ most popular shopping carts](http://www.shopping-cart-migration.com/supported-carts). Main beneficial sides for e-merchants are:

* No technical skills required. Since the process is fully automated, there is no need to have high level of programming skills. Intuitive [migration Wizard](http://www.shopping-cart-migration.com/quick-demo-tour) will help to perform a straightforward migration.
* Cart2Cart requires no software installation. The migration is performed on Amazon Elastic Cloud servers. The data migration is performed with the help of connection bridge, which is installed for source and target shopping carts and makes the integration possible.

* Cart2Cart provides a possibility to migrate products, customers,orders with all corresponding data to a desirable shopping cart.

* Flexible pricing system allows to pay only for entities, which are really migrated. The price of migration can be calculated with the help of [Online Estimator](https://app.shopping-cart-migration.com/estimator).

* Totally free Demo migration which provides a possibility to try how everything works before launching a Full migration.
If you have some questions, [Support Team](http://support.magneticone.com/index.php?/Tickets/Submit/RenderForm/5) is ready to answer all questions via live chat, phone or ticket at any convenient time.

= Do you migrate multiple languages? =
Yes, Cart2Cart supports migration of multiple languages from VirtueMart to WooCommerce providing a paid WPML module is installed and configured on your WordPress before the data transfer. Please, [read our FAQ](https://www.shopping-cart-migration.com/faq/45-woocommerce/385007-is-multilanguage-migration-to-from-woocommece-possible) for more info.

= Do you migrate customer passwords? =
Yes, there’s a possibility to migrate customer password from VirtueMart to WooCommerce by using a free Cart2Cart: Password Migration plugin. For more info, visit [this page](https://www.shopping-cart-migration.com/cart2cart-password-migration).

= Can I migrate product and category SEO URLs? =
Yes, you can migrate product and category SEO URLs from VirtueMart to WooCommerce by picking a corresponding additional option.

= Can I migrate Joomla users and Virtuemart customers? =
Our script distinguishes Joomla users and VirtueMart users. You can transfer customers of your VirtueMart store as a basic option and choose Joomla users migration - as additional. Because of CMS peculiarities, users from Joomla will be displayed as customers on your target store.

= Will the orders history be fully migrated for customer/administrator consultation? =
Sure, all previous orders will be migrated. You have to take into a consideration that all orders will have current status, as it isn't possible to transfer order status history.
= Can I try your service before setting up paid migration? =
Yes, we provide FREE Demo Migration service that you can use before starting full migration.
You should create an account and setup the source and target carts. You will be able to check the results of a limited number of entities (up to 10 products/customers/orders). Approximate time of Demo Migration is 10-30 minutes.
You can make any changes you need and run DEMO migration again. After you are satisfied with results you can run the live migration.
